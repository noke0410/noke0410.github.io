GitHub+Hexo的初體驗

發表於 2022-08-27 分類於 Hexo

前言

雖然很早以前就已經知道可以在 GitHub 建立自己的 Blog 站台，也許是不擅文字表達的關係，從來沒有想過要自己試著建立一個 Blog 去記綠這些年所學到的東西。直到最近開始嘗試，有一種彷彿又回到了新手村的感覺，不斷地嚐試與失敗。身為一個菜八巴，在參考了許多文章之後，決定跟隨多數人的腳步採用 GitHub+Hexo的架構進行 Blog 站台的架設。

想不到因為起步的太晚，所以參考到的許多文章都已經不敷使用，一下子不是這個不能用就是那個不能用的，兜兜轉轉的繞了好大一圈，還是回到官方網站參考文件最穩妥。果然人還是不能太偷懶，明明還不會爬就想飛，難怪會摔個鼻青臉腫的。光是為了最基礎的在 GitHub 上建立一個 Hexo 的 Blog 然後發佈一篇Hello World就花費了四個小時，更別提後續還要為 Hexo 挑選合適的主題，以及在 GitHub 上建立備份機制了。

Hexo

安裝需求

在安裝 Hexo 前必須先確認電腦已經安裝下列軟體：

  • Node.js
  • Git

安裝 Hexo

確認 Node.js 跟 Git 都已經安裝完畢之後就可以透過 npm 開始進行 Hexo 的安裝。

建立 Blog

安裝 Hexo 完畢後，可以參考官方文件執行下列指令，Hexo 會在指定資料夾中建立所有您需要的檔案。

依序執行到這一個步驟結束，基本上就已經建立好一個附帶Hello World最基礎的 Blog 了，接下來我們就可以進一步針對 _config.yml 進行網站的設定。

網站設定

依據官方文件所述，可以針對網站的 title(網站標題)、description(網站的描述)、keywords(網站的關鍵字)、language(網站使用的語言)、author(作者)、theme(網站使用的主題名稱)以及 deploy(佈署設定)等等諸多不同類型的屬性進行設定。

在 GitHub Pages 上部署 Hexo

一樣，我們可以參考官方文件進行設定。但在開始設定之前，必須先在 GitHub 建立好名為{username}.github.io 的 Repository，設定完畢之後就可以參考一鍵部署的部份去修改 _config.yml。

由於我希望除了 Blog 的功能之外，還要做到可以將整份 Hexo 備份到同一個 Repository，所以針對這個 Repository 的 Branch 做了一個簡單的規劃如下：

Branch 作用
master 備份 Hexo
gh-pages Blog 站台的檔案
